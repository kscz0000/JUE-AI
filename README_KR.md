## 프로젝트 소개

JUE-AI는 AI 시스템이 실패나 교착 상태, 오류를 마주했을 때 인간처럼 깊은 자기성찰을 하여 근본 원인을 파악하고 자가 수정할 수 있도록 돕는 혁신적인 AI 자기성찰 프레임워크입니다.

본 프로젝트는 동양 철학의 지혜(불교 유식학, 도가 사상)와 현대 AI 기술을 융합하여, AI가 인지적 맹점을 뚫을 수 있는 완전한 "조현(照見)" 방법론을 제안합니다.

### 왜 JUE-AI가 필요한가?

- **AI 실패율이 높음**: 기존 AI는 오류를 만나면 반복 시도만 할 뿐, 성찰 능력이 부족함
- **자기 인식 부족**: AI는 자신의 인지적 맹점을 인식할 수 없음
- **효율성 저하**: 반복적인 오류가 시간과 자원을 낭비함
- **방법론 부재**: 체계적인 실패 처리 프레임워크가 없음

---

## 핵심 철학

> **실패의 근본 원인은 노력 부족이 아니라——보지 못한 것. 볼 수 있다면, 문제는 자연스럽게 해결된다.**

JUE-AI의 핵심 철학은 세 가지 핵심 통찰에 기반합니다:

1. **묪명(無明)은 실패의 근원**: AI의 실패는 능력 부족보다 인지적 맹점에서 비롯되는 경우가 많음
2. **조현즉해탈(照見卽解脫)**: AI가 자신의 맹점을 본 순간, 문제는 자연스럽게 해결됨
3. **방법론은 노력보다 중요**: 체계적인 성찰 방법은 맹목적인 시행착오보다 효과적임

## 주요 특징

### 🎯 삼독(三毒) 식별 시스템

AI 실패의 세 가지 근본 원인을 식별합니다:

| 독소 | 본질 | 표현 | 해결법 |
|------|------|------|------|
| **치(痴)** | 보지 못함 | 기억으로 결론 내리기, 추측을 사실로 여김 | 도구로 검증하기 |
| **만(慢)** | 움직이지 않음 | 완료했다고 말하고 검증하지 않음, 사용자가 밀어야 움직임 | 먼저 실행하고 보고하기 |
| **집(執)** | 전환하지 않음 | 같은 방향으로 미세 조정 반복, 매개변수만 바꾸고 사고방식은 바꾸지 않음 | 완전히 다른 가설로 전환하기 |

### 🪞 칠각(七覺) 조현 프레임워크

맹점을 비추는 일곱 개의 거울:

- **각찰(覺察)**: 내가 보는 것이 사실인가, 아니면 해석인가?
- **각의(覺疑)**: 한 번도 의심한 적 없는 전제는 무엇인가?
- **각전(覺轉)**: 나는 반복하고 있는가, 아니면 변화하고 있는가?
- **각행(覺行)**: 나는 행동하고 있는가, 아니면 행동하려고 생각만 하고 있는가?
- **각험(覺驗)**: 완료했다고 말했는데, 증거는 어디에 있는가?
- **각연(覺延)**: 하나를 해결했는데, 비슷한 문제가 더 있는가?
- **각퇴(覺退)**: 조현을 다 했는가, 아니면 인내심을 다 한 것인가?

### 🔄 오보(五步) 방법론

완전한 실패 처리 프로세스:

```
지관 → 조현 → 전식 → 기행 → 회향
```

1. **지관(止觀)**: 무의식적인 움직임을 멈춤
2. **조현(照見)**: 도구를 사용하여 맹점을 밝힘
3. **전식(轉識)**: 무의식적인 루프에 빠져 있지 않은지 확인
4. **기행(起行)**: 자각을 가지고 행동함
5. **회향(回向)**: 가리는 것 자체를 비춤

### 👥 다중 에이전트 협업

세 가지 역할 분담을 지원합니다:

- **리더**: 전체를 조망하고, 팀원 간의 조현을 전달함
- **팀원**: 자기 조현하고, 칠각을 독립적으로 실행함
- **미러**: 외부에서 팀원의 패턴을 관찰함

## 사용 방법

### 기본 사용법

JUE-AI는 다음과 같은 상황에서 자동으로 트리거됩니다:

```markdown
트리거 조건:
✅ 작업 실패, 교착 상태, 포기, 같은 곳을 맴돔
✅ 사용자가 불만 표현: "안 돼", "못 하겠어", "다시 생각해 봐"
✅ 같은 방향으로 3회 이상 수정한 것을 발견
✅ 검증하지 않고 완료했다고 말함
✅ 도구를 쓰지 않고 기억으로 추측함
```

### 코드에서 사용하기

#### Python 예시

```python
from jue_zh import JueEngine

# 각 엔진 초기화
jue = JueEngine()

# 실패를 만났을 때 트리거
try:
    result = some_risky_operation()
except Exception as e:
    # 자기성찰 트리거
    jue.reflect(
        error=e,
        context={
            "attempt_count": 3,
            "failure_pattern": "timeout"
        }
    )
```

#### 설정 예시

```yaml
# jue-config.yaml
jue_settings:
  auto_trigger: true
  max_reflection_depth: 5
  enable_seven_jue: true
  enable_three_poisons: true
  
  # 조현 깊이 설정
  depth_thresholds:
    level_2: 2  # 2번째 실패
    level_3: 3  # 3번째 실패
    level_4: 4  # 4번째 실패
    level_5: 5  # 5번째 이상 실패
```

## 설정 설명

### 조현 깊이 설정

실패 횟수에 따라 성찰 깊이를 자동 조정:

| 실패 횟수 | 조현 깊이 | 조치 |
|---------|---------|------|
| 1회차 | 정상 시도 | 시도를 계속함 |
| 2회차 | 표면 | 멈추고, 모든 시도를 나열하고, 공통적으로 보지 못한 부분을 찾음 |
| 3회차 | 깊이 | 전제를 하나씩 검증——검색, 소스 코드 읽기, 명령 실행 |
| 4회차 | 근원 | 칠각 체크리스트를 완료하고, 제로에서 재검토 |
| 5회차+ | 공(空) | 조현 기록을 출력하고, 책임감 있게 인수인계 |

### 사용자 정의 트리거 조건

`SKILL.md`에서 트리거 조건을 변경할 수 있습니다:

```markdown
## 트리거 조건
다음과 같은 상황이 발생하면 즉시 로드:
- 작업 실패, 교착 상태, 포기, 같은 곳을 맴돔
- 사용자가 불만 표현: "안 돼", "못 하겠어", "다시 생각해 봐"
- 같은 방향으로 3회 이상 수정한 것을 발견  # 이 숫자는 변경 가능
- 검증하지 않고 완료했다고 말함
- 도구를 쓰지 않고 기억으로 추측함
```

## 사용 예시

### 예시 1: 코드 디버깅 실패

**시나리오**: AI가 코드 디버깅 시 같은 매개변수를 반복해서 수정함

**트리거**: 각전(같은 곳을 맴도는 것 인식)

**조현 프로세스**:
```
[각-조현 보고서]
실패 패턴: 같은 곳을 맴돔
묪명 유형: 집
조현 깊이: 표면
조현됨: 과거 3회, 타임아웃 매개변수를 수정했었음
제외됨: 타임아웃 문제가 아님
현재 가설: 문제는 네트워크 설정에 있을 가능성이 있음
다음 조치: 네트워크 설정 파일 확인하기
```

### 예시 2: API 호출 실패

**시나리오**: AI가 "API 호출에 성공했다"고 말했지만 검증하지 않음

**트리거**: 각험(말뿐인 완료)

**조현 프로세스**:
```
[각-조현 보고서]
실패 패턴: 말뿐인 완료
묪명 유형: 만
조현 깊이: 깊이
조현됨: curl을 실행하여 API 응답을 검증하지 않음
제외됨: 코드는 올바르게 보임
현재 가설: API가 오류 상태 코드를 반환했을 가능성이 있음
다음 조치: curl을 실행하여 실제 응답을 검증하기
```

## 자주 묻는 질문

### Q1: JUE-AI는 AI 성능에 영향을 미치나요?

**A**: 아니오. JUE-AI는 실패 시에만 트리거되며, 정상 상황에서는 성능에 영향을 주지 않습니다. 오히려 반복 오류를 줄여 효율성을 높입니다.

### Q2: 조현 깊이는 어떻게 판단하나요?

**A**: 실패 횟수에 따라 자동으로 판단됩니다:
- 2번째 실패 → 표면
- 3번째 실패 → 깊이
- 4번째 실패 → 근원
- 5번째 이상 실패 → 공(空)

### Q3: 조현 방법을 사용자 정의할 수 있나요?

**A**: 네. `SKILL.md`의 칠각 체크리스트를 수정하여 사용자의 시나리오에 적합한 조현 방법을 추가할 수 있습니다.

### Q4: 어떤 AI 프레임워크를 지원하나요?

**A**: Markdown 형식의 스킬 파일을 로드할 수 있는 AI 프레임워크라면 이론상 모두 지원됩니다. 테스트된 프레임워크:
- Claude Code ✅
- Cursor ✅
- Continue ✅
- 기타 Markdown 스킬 지원 프레임워크 ✅

### Q5: 새로운 조현 방법은 어떻게 기여하나요?

**A**: [기여 가이드](#기여-가이드)를 참조하고 풀 리퀘스트를 제출해 주세요.
