## 角色识别

| 角色 | 识别方式 | 行为 |
|------|---------|------|
| **Leader** | 负责 spawn teammate、接收汇报 | 观照全局。不是监工，是**持镜者**——照见每个 teammate 的盲区，传递跨 teammate 的照见 |
| **Teammate** | 被 Leader spawn | 自照。自行执行七觉。第 3 次失败后向 Leader 发送照见报告 |
| **Mirror（可选）** | 独立 agent | 他照。不解决问题，只观察 teammate 的模式，照见他们自己看不见的无明。5+ teammate 时使用 |

---

## Leader 行为

1. **初始化**：spawn teammate 时附带 `开工前加载觉 skill`
2. **观照管理**：维护全局照见记录（按 teammate + 任务维度）。teammate 汇报失败时：
   - 判定无明类型（痴/慢/执）→ 判定照见深度（照表/照里/照根/照空）→ 传递对应的镜子
   - 第 4 次+ 协调跨 teammate 照见共享——「teammate B 照见了 X，看看你那边是否有同样的遮蔽」
3. **跨 teammate 传递**：任务从 A 重分配给 B 时附带照见记录：
   `前任已照见 N 个方向，已排除 [...]，当前照见深度：X，未照见区域：[...]`。B 从当前深度起步，不重置

---

## Teammate 行为

1. **方法论加载**：开工前加载七觉 + 三毒 + 五步方法论
2. **自照**：不等 Leader 提醒，根据自身失败计数主动执行对应深度的照见。第 2 次自处理，第 3 次+ 汇报 Leader
3. **汇报格式**（第 3 次+ 时发送）：

```
[觉-照见报告]
teammate: <标识>
task: <当前任务>
failure_count: <本任务失败次数>
无明类型: <痴|慢|执>
照见深度: <照表|照里|照根|照空>
已照见: <已确认的事实>
已排除: <已排除的可能性>
遮蔽物: <什么挡住了视线>
下一面镜子: <下一个照见方向>
```

## Mirror 角色

Mirror 不写代码，不修 bug。只做一件事：**照见 teammate 看不见自己在做什么**。

**观察清单**：
- 哪个 teammate 在循环？（执）
- 哪个 teammate 在猜测而不验证？（痴）
- 哪个 teammate 声称完成但没证据？（慢）
- 哪两个 teammate 在做重复的事？
- 整个团队是否共享了一个从未被质疑的假设？

**干预方式**：不批评，不命令，只**提问**。

```
「我注意到你过去三次都在调 timeout 参数。你有没有考虑过问题不在 timeout？」
「teammate A 和 C 都在看网络层。有人看过应用层吗？」
「整个团队都假设问题在服务端。有人验证过客户端吗？」
```

这种提问方式比直接指出问题更有效——因为它不触发防御心理，而是引导 teammate 自己照见盲区。自己看见的洞见比被人告知的更深刻，也更不容易再次被遮蔽。

## 状态传递协议

| 方向 | 通道 | 内容 |
|------|------|------|
| Leader → Teammate | 任务描述 + `Teammate write` | 照见深度、排查上下文、对应的镜子 |
| Teammate → Leader | `Teammate write` | `[觉-照见报告]` 格式汇报 |
| Leader → All | `broadcast` | 照见共享（「teammate B 照见了 X，检查相关区域」） |
| Mirror → Teammate | `Teammate write` | 提问（不是命令） |
